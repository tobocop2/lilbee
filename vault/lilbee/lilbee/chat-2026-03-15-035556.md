## Core Functionality

lilbee is a local AI knowledge base plugin for Obsidian that allows you to search, ask questions, and chat with your vault's content using local LLMs via Ollama.

## Setup Process

1. **Install prerequisites**:
   - Install [Ollama](https://ollama.com) (local LLM runtime)
   - Install lilbee: `pip install lilbee` or `uv tool install lilbee`

2. **Start services**:
   - Start Ollama: `ollama serve`
   - Start lilbee server in your vault directory:
     ```bash
     cd /path/to/your/vault
     lilbee init
     lilbee serve
     ```

3. **Install plugin**:
   - Copy `main.js`, `manifest.json`, and `styles.css` into `.obsidian/plugins/lilbee/`
   - Enable "lilbee" in Obsidian's community plugins settings

## Key Features

### Search
- **Instant semantic search** across indexed vault content
- Real-time results with 300ms debounce
- Results grouped by document with relevance-ranked excerpts showing page numbers and line ranges
- Click source filenames to open documents in Obsidian

### Ask
- Natural language questions answered with synthesized responses from vault content
- Source citations included with answers

### Chat
- Multi-turn conversation with knowledge base
- Streaming token-by-token responses with markdown rendering
- Full conversation history within session
- Source citations with expandable details
- Paperclip button to add files from vault or filesystem
- Model selector dropdown with curated catalog and installed models

### Sync
- Indexes vault documents into `.lilbee/` vector database
- Hash-based change detection (only re-indexes changed files)
- Progress tracking during indexing
- Auto-sync mode available (watch for file changes and sync automatically)

## Supported Formats

lilbee indexes various document formats including:
- PDF (extracts embedded text; falls back to OCR for scanned pages)
- Office documents (.docx, .xlsx, .pptx)
- eBooks (.epub)
- Images (.png, .jpg, .jpeg, .tiff, .bmp, .webp) - requires vision model for OCR
- Data files (.csv, .tsv)
- Structured data (.xml, .json, .jsonl, .yaml, .yml)
- Text files (.md, .txt, .html, .rst)
- Code files (.py, .js, .ts, .go, .rs, .java, and 150+ more) with AST-aware chunking

## Model Requirements

The first-time download of Ollama models is required (e.g., `qwen3:8b` is ~5 GB, `nomic-embed-text` is ~274 MB). After initial download, models are cached locally and load in seconds. The embedding model (`nomic-embed-text`) is auto-pulled on first sync.

## Commands

All commands can be accessed via Obsidian's command palette (Ctrl/Cmd + P):
- "lilbee: Search knowledge base"
- "lilbee: Ask a question" 
- "lilbee: Open chat"
- "lilbee: Sync vault"
- "lilbee: Add current file"
- "lilbee: Add current folder"
- "lilbee: Show status"

The plugin requires Ollama to be running for sync operations, as it needs the embedding model.
