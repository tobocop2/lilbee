## Did the Agents Follow Instructions?

Both [AGENTS.md](../../demos/godot-with-lilbee/AGENTS.md) files use the same 4-step workflow: plan â†’ search each class/method â†’ write using only confirmed APIs â†’ verify every call before saving.

**The lilbee version followed instructions well.** It called `search` multiple times during planning and verification. All Godot API calls are correct â€” the only issue is a structural one (uninitialized `tile_map`), not an API hallucination.

**The without-lilbee version partially followed instructions.** It used `codesearch` but still produced 4 API errors â€” `heuristic` (should be `default_compute_heuristic`), `grow_individual(1)` (needs 4 args), `rng.bool()` (doesn't exist), and a type mismatch on `get_id_path`. These suggest the agent searched but didn't verify individual method signatures as step 4 requires.

## What RAG Changed

**API accuracy.** The lilbee version has zero incorrect Godot API calls. Every property, method, and enum maps exactly to the [Godot 4.4 class reference][godot-docs]. The without-lilbee version has 4 incorrect calls â€” `heuristic`, `grow_individual(1)`, `rng.bool()`, and a return type mismatch. Web search found the right classes but missed signature details.

**Both found `AStarGrid2D`.** Unlike the previous recording where the no-lilbee version fell back to `AStar2D`, this time both versions used `AStarGrid2D`. The stronger AGENTS.md (explicitly naming `codesearch`) helped the baseline agent search more thoroughly.

**Hallucination rate.** With lilbee: 0 hallucinated APIs out of ~15 distinct calls. Without lilbee: 4 hallucinated APIs out of ~18 distinct calls (~22% error rate). The indexed class reference XML gave the agent exact method signatures, while web search results were close but imprecise.

## Room for Improvement

These demos used a single prompt with no follow-ups and a minimal [AGENTS.md](../../demos/godot-with-lilbee/AGENTS.md). Several things could improve results:

- **Multi-turn iteration** â€” let the agent run the script in Godot, see errors, and fix them
- **Better model** â€” minimax-m2.5-free is a free-tier cloud model; a stronger model may follow the verify step more faithfully
- **More targeted indexing** â€” index only the ~20 classes relevant to 2D tile-based games instead of all 917, reducing noise in search results
- **Stronger AGENTS.md** â€” require the agent to print search results and confirm each method signature before writing code
